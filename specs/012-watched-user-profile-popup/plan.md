# Implementation Plan: Watched User Profile Popup

**Branch**: `012-watched-user-profile-popup` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/012-watched-user-profile-popup/spec.md`

## Summary

Add a floating profile popup that opens when `<CR>` is pressed on a watched username — either from the "Watched Users" section header rows in the dashboard, or from the manager popup (`<leader>gu`). The popup shows the user's public profile stats and 26-week contribution heatmap, fetched on-demand using the existing `gh` CLI and GraphQL patterns.

## Technical Context

**Language/Version**: Lua (LuaJIT 2.1), Neovim 0.12.0-dev  
**Primary Dependencies**: `gh` CLI v2.45.0 (REST + GraphQL), `vim.system()` (async), `nvim_open_win` (floating windows)  
**Storage**: N/A — no persistence; data fetched fresh on each popup open  
**Testing**: Manual smoke test  
**Target Platform**: Neovim on Linux (WSL2)  
**Project Type**: Neovim plugin (Lua modules)  
**Performance Goals**: Popup window opens immediately; content renders within 3 seconds  
**Constraints**: Two async fetches (profile + contributions) fan out in parallel, render when both complete  
**Scale/Scope**: Single popup per open; no concurrent popup management needed

### Modifications to `github_dashboard.lua`

#### `render_watched_users` — group events by actor

Change the event rendering loop to:
1. Build ordered actor list (preserving first-appearance order from sorted events)
2. For each actor:
   - Insert actor header line: `"   @username"` with `GhUsername` highlight
   - Insert item: `{ line=#lines, kind="user", username=actor }`
   - For each of that actor's events: render indented event row (same format as now, but drop actor column — show just icon + repo + age)

**New event row format** (actor is now in the header):
```
"      %s  %-32s  %s"  -- icon, repo, age (indented 6 spaces)
```

#### `open_url_at_cursor` — route `kind="user"` to profile popup

```lua
if item.kind == "issue" or item.kind == "pr" or item.kind == "repo" then
  require("alex.gh_reader").open(item)
elseif item.kind == "user" then
  require("alex.gh_user_profile").open(item.username)
else
  vim.system({ "xdg-open", item.url })
end
```

### Modifications to `gh_user_watchlist.lua`

#### `open_manager` — add `<CR>` keymap and update footer

In the `bmap` section:
```lua
bmap("<CR>", function()
  if not state.manager_win or not vim.api.nvim_win_is_valid(state.manager_win) then return end
  local cur = vim.api.nvim_win_get_cursor(state.manager_win)[1]
  local idx = cur - 1  -- line 1 = "", line 2 = first user
  if idx >= 1 and idx <= #state.users then
    require("alex.gh_user_profile").open(state.users[idx])
  end
end)
```

Footer update:
```
" a add  ·  d remove  ·  <CR> profile  ·  q close "
```
